INSTRUCCIONES PARA LA EJECUCION DEL PROGRAMNA

EJECUCIÓN DEL PROGRAMA
1-INICIALIZA EL CMD PARA LA EJECUCION DE LA TAREA POR MEDIO DE LA HERRAMIENTA DE TASK WINDOWS.
2-ABRE EL NAVEGADOR DE MANERA AUTOMATICA, CONFIGURANDO SUS PREFERENCIAS DE DESCARGA.
3-ABRE LA NUBE DE DIACLOUD Y EJECUTA LAS CREDENCIALES.
4-NAVEGA POR LAS SECCIONES DE LA PAGINA EN BUSCA DEL APARTADO DE REGISTRO.
5.INICIA EL PROCESO DE DESCARGA DE DATOS: HISTORIAL,EXPORTAR Y CERRAR REGISTRO.
6-PROCESA LOS ARCHIVOS Y LOS MANDA A LA BASE DE DATOS.

INSTRUCCIONES PARA LA CONFIGURACION DEL PROGRAMA

SE ENCUENTRAN 3 CARPETAS QUE RESPECTAN AL PROCESAMIENTO DEL PROGRAMA. EL IMPORTANTE A CONFIGURAR PARA LA EJECUCION DEL MISMO ES LA CARPETA "EJECUCION DEL PROGRAMA", EN ESTE APARTADO SE ENCUENTRA EL PROGRAMA EN VISUAL STUDIO, SE DEBE DE ABRIR LA CARPETA DE BD. POR EL HECHO DE QUE SE ENCUENTRAN CADA MODULO EN ESE APARTADO.

EN ESA MISMA CARPETA TAMBIEN SE ENCUENTRAN DOS ARCHIVOS POR LOTES DE WINDOWS, SON LOS QUE ESTAN CON ENGRANAJE. ESTOS NOS PERMITEN EJECUTAR EL PROGRAMA Y LA TAREA DE WINDOWS. La carpeta para la configuración de la tarea es con respecto al archivo "ejecutar_programa".

Ejecutar_programa CONFIGURACION

1-Opciones de Seguridad
	1.1 Ejecutar solo cuando el usuario haya iniciado sesión.
	1.2 Ejecutar con los priveligios mas altos.

2-Desencadenador
	2.1 Iniciar la tarea: Al iniciar Sesión.
	2.2 Cualquier usuario
	2.3 Repetir cada 1 hora
	2.5 Durante indefinidamente
	2.6 Habilitador

3-Accion
	3.1 Iniciar un programa
	3.2 Agregar la ruta del archivo de "ejecutar_programa"

4-Condiciones 
	4.1 Iniciar la tarea solo si el equipo esta conectado a la corriente
	4.2 Detener si el equipo empieza a usar la batería
	4.3 Iniciar solo si la conexión de red esta disponible. "cualquiera".

5-Configuraciones
	5.1 Permitir que la tarea se ejecute a petición
	5.2 Ejecutar tarea lo antes posible si no hubo inicio programa
	5.3 Si la tarea no se ejecuta, reiniciarla cada 1 minuto.(cambiarlo si es que es 	necesario).(INTENTAR REINICIO UN MAX DE 3 VECES).
	5.4 Detener tarea en ejecución si no finaliza cuando se solicite
	5.5 No iniciar una instancia nueva.

CIA_CLOUD

ESTE MISMO SE ENCARGA DE LA EJECUCION DEL PROGRAMA DE VISUAL STUDIO SOLO VERIFICAR SI ES NECESARIO SU RUTA, ES DECIR SI ESTA REFERIDA DENTRO DEL PROGRAMA DE VISUA. VERIFICACION POR EJECUCION. SI ES QUE NO SE INCIA EL PROGRAMA ENTONCES SI SERA NECESARIO REVISAR LA RUTA DE CIA_CLOUD.(NO ES NECESARIO CAMBIAR NADA MAS QUE LA RUTA.)


BASE DE DATOS 

LA BASE DE DATOS CUENTA CON UN USARIO Y UNA CONTRASEÑA, SE ENCUENTRA EN EL ARCHIVO DE TXT DE LA CARPETA DE EJECUCION DE PROGRAMA "CONEXION A LA BASE DE DATOS".
EN ESTA CARPETA SE ENCUENTRA EL PROGRAMA EN MYSQL DONDE EN ESTE MISMO SE ENCUENTRA LA PROGRAMACION DE LA MISMA.
TAMBIEN UN SCRIPT DE CONSULTAS EN CASO DE QUE SEA NECESARIO.
POR OTRO LADO TAMBIEN ESTA EL ARCHIVO DE POWERBI PARA VISUALIZAR LOS DATOS DE UNA MANERA MAS COMODA.(NOTA LA CONFIGURACION DEL POWERBI TIENE PROBLEMAS PARA LA CONEXION DE BASES CON MYSQL, SE TE SOLICITARA QUE DESCARGUES CIERTOS RECURSOS PARA REALIZAR LA CONEXION. DESCARGAR LAS VESIONES 8, SON LAS QUE MEJOR FUNCIONAN POR EL HECHO DE QUE LAS VERSIONES ACTUALES NO PERMITEN LA CONEXION.)


NOTA EN GENERAL

EL PROGRAMA PUEDE NO EJECUTARSE DESDE UN PRINCIPIO POR EL MOTIVO DEL CAMBIO DE RUTAS.
VERIFICAR LAS RUTAS NECESARIAS.

RUTAS A VERIFICAR

1-RUTA DE BD(CARPETA DONDE SE ENCUENTRAN LOS MODULOS DEL PROGRAMA)
2-RUTA DE LA CARPETA DE DESCARGA DE DATOS.(SE DEBE CAMBIAR DENTRO DEL PROGRAMA,REVISAR LAS CARPETAS MAIN Y PROCESAMIENTO DE DATOS).
3-RUTA DE CIA_CLOUD
4-RUTA DE ejecutar_programa
5-RUTA DE BASE DE DATOS DE MYSQL, AUNQUE POSIBLEMENTE NO SEA NECESARIO POR MOTIVO DE QUE PARA LA CONEXION ESTA POR USUARIO Y CONTRASEÑA. REVISAR LA CONEXION CON LA BASE DENTRO DE VISUAL STUDIO CODE. Si se necesita cambiar la ruta realizarla.


SI EN CASO DE QUE SEA REQUERIDO CAMBIAR PARTE DEL CODIGO, TRABAJA EN UN ARCHIVO DE RESPALDO, EL PROGRAMA ESTA EJECUTADO POR MODULOS PARA FACILITAR EL PROCESAMIENTO DE CAMBIOS Y EVITAR UN ERROR GENERAL EN EL PROGRAMA, OBVIAMENTE SI SE REALIZA UN CAMBIO PUEDE AFECTAR A LOS DEMAS MODULOS POR ESTAR INTERCONECTADOS, REVISA BIEN EL PROGRAMA PARA OPTIMIZAR EL FLUJO DE EJECUCION DEL PROGRAMA.
